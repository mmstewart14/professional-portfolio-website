Machine Learning Fitness App Project

BACKGROUND

This project was for an Introduction to Machine Learning Class at OSU. For our final project, we were given the broad of using an ML algorithm to solve a problem. I decided to make an app, which uses ML to generate user-specific random weight lifting routines for the user, as I had recently learned how beneficial it can be for strength gains to change your routine frequently. The app uses two ML algorithms, a supervised learning Random Forest Classification and a Reinforcement Learning Greedy Algorithm. The app was hosted using Python's Flask, using Python, HTML, CSS, and Javascript.

DESIGN

Since the app needs to keep track of different data for each user, user accounts, and thus, a login screen was needed.

Login Screen

When using the app for the first time, a user would need to provide some labeled data to the SL Random Forest (RF) algorithm. This was accomplished on the following screen. The app would display randomly generated workouts, and a user would indicate whether or not they liked each. The app would continue to show random workouts until a user had picked at least 5 liked plans and 5 disliked plans, in order to ensure the RF was provided with enough training & testing data.

Training Screen

Once the RF algorithm was trained and tested, the app can provide its main functionality: generating daily user-specific random workout plans. The following screen displays these plans. The ML continues here, both algorithms actually. If a user does not like the generated plan, they can press the button at the bottom to let the app know they do not like it and generate a new plan. Each time the button is pressed, the disliked plan is added to the RF training & testing data. It is also assigned a negative weight, so that the greedy algorithm is less likely to chose that plan in the future.

Main Daily Workout Screen

BACK-END

  • Scripted ML algorithms using scikit-learn library.
  • Classified newly generated workout plans, so as to only display predicted liked plans to the user.
  • Generated new workout plans from a master CSV file.
  • Served workout data from Python to JavaScript to be displayed on front-end.
